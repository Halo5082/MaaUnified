# MAAUnified 回滚手册 v1

## 触发条件
- `ci-avalonia.yml` 任一阻断 gate 失败且短期无法修复。
- 线上主流程阻断（连接、启动、停止或导入）导致不可发布。
- 平台关键能力异常且降级路径不满足“可见、可记录、可定位”。
- 发布产物完整性校验失败（缺少 runtime 或 resource）。

## 回滚步骤
1. 冻结当前发布流程，停止继续分发新产物。
2. 在发布渠道撤回当前 `MAAUnified` 版本引用。
3. 切换到上一个已验证版本产物（按平台分别回退）。
4. 保留并归档当前版本的失败证据：CI日志、TRX、关键 debug 日志。
5. 对外更新状态说明，标记回滚窗口与影响范围。

## 回滚后验收
- 旧版本前端可正常启动。
- 旧版本可以完成一次 `连接 -> 执行 -> 停止` 全流程。
- 配置读取正常，关键设置项无破坏性变化。
- 平台降级路径仍满足“可见、可记录、可定位”。
- 发布渠道仅暴露回滚后的稳定产物。

## 数据保留策略
- 保留 `config/avalonia.json`，不做自动删除或覆写。
- 保留 `debug/avalonia-ui-errors.log`、`debug/avalonia-platform-events.log`、`debug/config-import-report.json` 供复盘。
- 失败版本的 CI TRX 与构建日志至少保留一个发布周期。
- 回滚过程中禁止执行会破坏用户数据的批量清理动作。

## 责任人与响应时限
- Owner: `MAAUnified Release Owner`
- Backup owner: `MAAUnified QA Owner`
- 响应时限：
  - `P0` 发布阻断：15 分钟内确认回滚决策，60 分钟内完成回滚。
  - `P1` 非阻断风险：4 小时内完成风险评估与处置结论。
- 回滚完成后 24 小时内输出复盘结论与修复计划。
